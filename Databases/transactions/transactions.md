-- 1.1.1. RegisterUser
-- Косая запись не мешает, так как read only.
-- Фантомная запись не мешает, так как читаем не более одного раза.
-- Неповторяемое чтение не мешает, так как читаем не более одного раза.
-- Грязное чтение мешает, так как грязные данные могут помешать создать пользователя.
start transaction read write isolation level read committed;

-- 1.1.2. ManageFlight
-- Косая запись не мешает, так как функция не может нарушить какие-либо инварианты.
-- Фантомная запись не мешает, так как добавление новых строк в таблицах никак не влияет на работу функции (тк задача в модификации настроек).
-- Неповторяемое чтение мешает, так как после проверки на админа не хотелось бы, чтобы права изменились (или пароль) + для корректной работы RefreshParams (проверки на время).
start transaction read write isolation level read repeatable;
